# Support Boundaries

## Standard Response Expectations

Standard engagements are designed around planned work and 24-48 hour response
expectations, not same-day emergency managed service unless explicitly
contracted.

## Emergency Support

Emergency support is a separate agreement. It needs defined coverage hours,
access, severity levels, response expectations, and billing.

## What Is Included

Typical included work:

- scoped deployment
- basic documentation
- verification
- handoff notes
- planned updates within scope
- agreed troubleshooting during the engagement

## What Is Not Included

Unless explicitly contracted, support does not include:

- 24/7 emergency response
- unlimited app customization
- unlimited migration work
- legal or compliance advice
- guaranteed email deliverability
- high availability
- custom application development
- unmanaged third-party client devices

## Client Responsibilities

The client owns:

- domain/DNS access
- server access
- admin decision-maker
- backup destination
- account ownership
- timely responses
- acceptance testing

## Change Requests

Change requests should identify the business reason, affected users, services,
data migration needs, rollback expectations, and testing criteria.

## Maintenance Windows

Updates and service changes should use planned maintenance windows when they
can affect user access, data migrations, mail, identity, or shared storage.

## Handoff

Handoff should include deployed service URLs, admin ownership, backup notes,
verification results, known caveats, and the next recommended maintenance step.
