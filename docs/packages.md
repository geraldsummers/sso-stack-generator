# Packages

## Package Comparison

| Package | Best for | Includes | Excludes |
| --- | --- | --- | --- |
| Secure Core | tiny teams/private operators | SSO, docs, passwords, portal, backups, basic monitoring | mail, AI/data, heavy migration |
| Ops Platform | small teams | files, projects, chat, mail/calendar, Git, observability | advanced AI/data workflows |
| AI Sovereign Lab | technical teams | Jupyter, workspaces, OpenSearch, Qdrant, connector tooling | custom model hosting unless scoped |

## Secure Core

Secure Core is for owners, founders, private operators, tiny teams,
households, and small orgs that need a clean private base.

Typical scope:

- Caddy
- Keycloak
- service portal/catalog
- Vaultwarden
- BookStack
- basic monitoring
- backups
- verification

## Ops Platform

Ops Platform is for small teams replacing scattered SaaS tools with a coherent
client-owned platform.

Typical scope:

- everything in Secure Core
- files
- project boards
- mail/calendar/contacts
- chat
- Git hosting
- deeper observability
- operational docs

## AI Sovereign Lab

AI Sovereign Lab is for technical teams, data-heavy operators, and
agent-assisted workflows.

Typical scope:

- everything in Ops Platform
- JupyterHub
- disposable workspaces
- OpenSearch and Qdrant
- ChatGPT connector/MCP tooling
- knowledge ingestion and search workflows where applicable

## Common Inclusions

- deployment
- basic documentation
- generated config
- auth integration
- verification
- handoff notes

## Common Exclusions

- formal compliance certification
- legal advice
- 24/7 emergency support unless contracted
- unlimited app customization
- unlimited data migration
- guaranteed email deliverability
- high availability unless explicitly designed

## How To Choose

Choose Secure Core when the main problem is ownership, identity, docs,
passwords, and basic operations.

Choose Ops Platform when the team wants to replace several internal SaaS tools
with a shared private workspace.

Choose AI Sovereign Lab when the team already has technical workflows that need
private notebooks, workspaces, search backends, or connector tooling.

## Scope Boundaries

Package selection defines the starting surface. Data migration, custom app
development, high availability, compliance evidence, emergency support, and
model hosting are separate scopes unless explicitly included.
